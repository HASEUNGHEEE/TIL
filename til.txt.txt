2. 1주차 주요 개념 키워드 적어보기
1) 버전관리 = 프로젝트 상태가 변경되는 정보를 알고 있다는 것
2) 순서: git 초기화(git initialize) -> add(staging) -> commit
3) git 초기화: 기존의 프로젝트를 git 프로젝트로 만든다.
4) add: commit에 반영할 파일을 선택하는 것
5) commit: 현재 프로젝트의 상태를 찰칵 저장하는 스냅샷
6) commit에는 누가, 언제, 어떤 작업을 했는지에 대한 정보와, commit 아이디가 포함된다.
7) 원격 repo와 로컬 repo를 연결해서 내용을 반영하고 싶을 때
	- clone: 로컬 repo에 없고 원격 repo에 있는 내용을 가져오고 싶을 때
	- 로컬 repo가 원격 repo를 tracking 하도록 설정: 원격 repo와 로컬 repo에 둘 다 있고 두 개를 연결하고 싶을 떄
